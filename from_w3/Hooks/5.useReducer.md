`useReducer` is a React Hook used to manage **complex state logic** in a predictable way. Think of it as an alternative to `useState` when:

* State has multiple related values.
* State updates depend on previous state.
* Many different actions can change the state.

## Basic Idea

With `useState`:

```jsx
const [count, setCount] = useState(0);
```

With `useReducer`:

```jsx
const [state, dispatch] = useReducer(reducer, initialState);
```

Where:

* **state** → current state value
* **dispatch** → function used to send actions
* **reducer** → function that decides how state changes
* **initialState** → starting state

---

# Real-Life Analogy

Imagine a bank account.

* Current balance = **state**
* Deposit/withdraw request = **action**
* Bank rules = **reducer**
* Bank employee processing requests = **dispatch**

You don't directly change the balance. You send an action, and the reducer decides the new balance.

---

# Syntax

```jsx
const [state, dispatch] = useReducer(reducer, initialState);
```

Example reducer:

```jsx
function reducer(state, action) {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };

    case "decrement":
      return { count: state.count - 1 };

    default:
      return state;
  }
}
```

# Complete Example

```jsx
import React, { useReducer } from "react";

const initialState = {
  count: 0,
};

function reducer(state, action) {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };

    case "decrement":
      return { count: state.count - 1 };

    case "reset":
      return initialState;

    default:
      return state;
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <div>
      <h2>{state.count}</h2>

      <button onClick={() => dispatch({ type: "increment" })}>
        +
      </button>

      <button onClick={() => dispatch({ type: "decrement" })}>
        -
      </button>

      <button onClick={() => dispatch({ type: "reset" })}>
        Reset
      </button>
    </div>
  );
}

export default Counter;
```

### Flow

1. User clicks `+`
2. `dispatch({ type: "increment" })`
3. Reducer receives action
4. Reducer returns new state
5. React re-renders component

# Understanding the Reducer Function

A reducer always receives:

```jsx
function reducer(state, action) {
  // logic
}
```

### state

Current state:

```jsx
{
  count: 5
}
```

### action

Information about what happened:

```jsx
{
  type: "increment"
}
```

or

```jsx
{
  type: "add",
  payload: 10
}
```

# Using Payload

Sometimes you need to send data along with the action.

```jsx
function reducer(state, action) {
  switch (action.type) {
    case "add":
      return {
        count: state.count + action.payload,
      };

    default:
      return state;
  }
}
```

Dispatch:

```jsx
dispatch({
  type: "add",
  payload: 5,
});
```

Result:

```jsx
count + 5
```

# Managing Multiple State Values

With `useState`:

```jsx
const [name, setName] = useState("");
const [age, setAge] = useState(0);
const [city, setCity] = useState("");
```

With `useReducer`:

```jsx
const initialState = {
  name: "",
  age: 0,
  city: "",
};
```

Reducer:

```jsx
function reducer(state, action) {
  switch (action.type) {
    case "setName":
      return {
        ...state,
        name: action.payload,
      };

    case "setAge":
      return {
        ...state,
        age: action.payload,
      };

    case "setCity":
      return {
        ...state,
        city: action.payload,
      };

    default:
      return state;
  }
}
```

Usage:

```jsx
dispatch({
  type: "setName",
  payload: "John",
});
```

# Todo Example

```jsx
const initialState = {
  todos: [],
};

function reducer(state, action) {
  switch (action.type) {
    case "ADD_TODO":
      return {
        todos: [...state.todos, action.payload],
      };

    case "DELETE_TODO":
      return {
        todos: state.todos.filter(
          todo => todo.id !== action.payload
        ),
      };

    default:
      return state;
  }
}
```

Add todo:

```jsx
dispatch({
  type: "ADD_TODO",
  payload: {
    id: Date.now(),
    text: "Learn useReducer",
  },
});
```

Delete todo:

```jsx
dispatch({
  type: "DELETE_TODO",
  payload: 123,
});
```

# useState vs useReducer

| useState            | useReducer           |
| ------------------- | -------------------- |
| Simple state        | Complex state        |
| Easy to learn       | Slightly more setup  |
| Few updates         | Many update types    |
| Small components    | Large components     |
| Direct state update | Action-based updates |

### Good useState example

```jsx
const [count, setCount] = useState(0);
```

### Good useReducer example

```jsx
const [state, dispatch] = useReducer(reducer, initialState);
```

when state has:

```jsx
{
  users: [],
  loading: false,
  error: null
}
```

# Common Pattern

For API calls:

```jsx
const initialState = {
  loading: false,
  data: null,
  error: null,
};
```

Reducer:

```jsx
function reducer(state, action) {
  switch (action.type) {
    case "FETCH_START":
      return {
        ...state,
        loading: true,
      };

    case "FETCH_SUCCESS":
      return {
        loading: false,
        data: action.payload,
        error: null,
      };

    case "FETCH_ERROR":
      return {
        loading: false,
        data: null,
        error: action.payload,
      };

    default:
      return state;
  }
}
```

This keeps all related state updates in one place.

# Important Rules

### 1. Never mutate state

❌ Wrong

```jsx
state.count++;
return state;
```

✅ Correct

```jsx
return {
  count: state.count + 1,
};
```

---

### 2. Always return a state

❌ Wrong

```jsx
case "increment":
  state.count++;
```

✅ Correct

```jsx
case "increment":
  return {
    count: state.count + 1,
  };
```

---

### 3. Use a default case

```jsx
default:
  return state;
```

# Mental Model

```text
User Action
      ↓
dispatch(action)
      ↓
Reducer
(state, action)
      ↓
Returns New State
      ↓
React Re-renders
```

### Quick Summary

`useReducer` = **state + action + reducer**

* **State** = current data
* **Action** = what happened
* **Dispatch** = sends action
* **Reducer** = calculates new state

Use `useReducer` when state logic becomes difficult to manage with multiple `useState` calls. For simple counters, inputs, and toggles, `useState` is usually enough.
