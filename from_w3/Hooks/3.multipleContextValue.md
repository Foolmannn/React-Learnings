In React, `useContext` can provide **any JavaScript value**—an object, array, string, number, function, or a combination of these. The most common approach for passing multiple values is to pass an **object** as the context value.

## 1. Passing Multiple Values Using an Object (Recommended)

### Create Context

```jsx
import { createContext } from "react";

export const UserContext = createContext();
```

### Provide Multiple Values

```jsx
import { UserContext } from "./UserContext";

function App() {
  const username = "Suman";
  const age = 23;

  const login = () => {
    console.log("Logged in");
  };

  return (
    <UserContext.Provider
      value={{
        username,
        age,
        login
      }}
    >
      <Dashboard />
    </UserContext.Provider>
  );
}

export default App;
```

### Consume Values

```jsx
import { useContext } from "react";
import { UserContext } from "./UserContext";

function Dashboard() {
  const { username, age, login } = useContext(UserContext);

  return (
    <>
      <h1>{username}</h1>
      <p>{age}</p>
      <button onClick={login}>Login</button>
    </>
  );
}
```

## 2. Passing State and State Updater

A very common pattern:

```jsx
import { createContext, useState } from "react";

export const ThemeContext = createContext();

function ThemeProvider({ children }) {
  const [theme, setTheme] = useState("light");

  return (
    <ThemeContext.Provider
      value={{
        theme,
        setTheme
      }}
    >
      {children}
    </ThemeContext.Provider>
  );
}
```

Using it:

```jsx
const { theme, setTheme } = useContext(ThemeContext);

setTheme("dark");
```

## 3. Passing Multiple States

```jsx
<UserContext.Provider
  value={{
    user,
    setUser,
    isLoggedIn,
    setIsLoggedIn,
    theme,
    setTheme
  }}
>
  {children}
</UserContext.Provider>
```

Consume:

```jsx
const {
  user,
  setUser,
  isLoggedIn,
  setIsLoggedIn,
  theme,
  setTheme
} = useContext(UserContext);
```

---

## 4. Passing an Array Instead

You can also pass an array:

```jsx
<UserContext.Provider value={[user, setUser]}>
  {children}
</UserContext.Provider>
```

Consume:

```jsx
const [user, setUser] = useContext(UserContext);
```

This is similar to how `useState` returns values.

---

## 5. Example: Global User Context

```jsx
import { createContext, useState } from "react";

export const UserContext = createContext();

export function UserProvider({ children }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(false);

  const logout = () => {
    setUser(null);
  };

  return (
    <UserContext.Provider
      value={{
        user,
        setUser,
        loading,
        setLoading,
        logout
      }}
    >
      {children}
    </UserContext.Provider>
  );
}
```

Usage:

```jsx
const {
  user,
  loading,
  logout
} = useContext(UserContext);
```

## Why Objects Are Preferred

Suppose you use an array:

```jsx
value={[user, loading, logout]}
```

Then consuming code must remember the order:

```jsx
const [user, loading, logout] = useContext(UserContext);
```

If you later add another value, all consumers may break.

With an object:

```jsx
value={{ user, loading, logout }}
```

Consumers can access only what they need:

```jsx
const { user, logout } = useContext(UserContext);
```

This is more readable, maintainable, and is the standard pattern in most React applications.
